# 설계 결정 기록

## 2026-02-26 (최신)
- [D-021] 3D 뷰어에 `renderMode: tower | ray` 분기 구조를 추가하고 기존 타워 렌더링을 유지.
  - 이유: 기존 그래프형 표현을 보존하면서 천장 설치 시나리오의 Ray 시각화를 병행 제공하기 위해.
- [D-022] Ray 모드 렌더링을 `LineSegments(64개 레이)` + `Points(64개 끝점)`의 BufferGeometry 갱신 방식으로 구현.
  - 이유: 프레임마다 geometry 재생성 없이 `position/color` attribute만 업데이트해 성능을 유지하기 위해.
- [D-023] Ray 계산 기준을 `sensorPos=(0, ceilingHeight, 0)` + FOV 기반 사전 계산 `rayDir[64]`로 고정.
  - 이유: 천장 설치 모델을 명확히 하고 매 프레임 계산량을 줄이기 위해.
- [D-024] Ray 모드 색상/범례 기준을 depth(mm)에서 바닥 기준 높이 `h(mm)=p.y*1000`으로 전환.
  - 이유: 사용자 목적(높이 관찰/측정)에 직접 대응되는 값을 UI에 표시하기 위해.
- [D-025] 카메라 프리셋을 `topdown` 단일로 단순화하고 페이지 기본 렌더 모드를 `ray`로 변경.
  - 이유: 운영자가 주로 사용하는 뷰를 기본값으로 두고 조작 복잡도를 줄이기 위해.
- [D-026] Ray `topdown` 뷰는 상부 사선 시점으로 운용하며 상하각은 고정, 좌우 회전은 360도 허용.
  - 이유: 높이 판독 안정성과 주변 확인용 회전 자유도를 동시에 확보하기 위해.
- [D-027] Ray 모드의 바닥 표현을 바둑판(grid)에서 반투명 바닥 평면으로 변경.
  - 이유: 실제 방바닥 느낌을 주면서 시각적 잡음을 줄이기 위해.
- [D-019] 3D 뷰어 기본 카메라를 한 단계 확대(FOV 축소 + 거리 단축)로 조정.
  - 이유: 사용자 요청에 맞춰 8x8 변화 영역을 더 크게 관찰하기 위해.
- [D-020] 루트 `.gitignore`를 추가해 `node_modules/.next/webapp-buildcheck/*.tsbuildinfo`를 제외하고 원격 저장소에 푸시.
  - 이유: 저장소 비대화/불필요 산출물 커밋을 방지하고 배포 가능한 소스만 관리하기 위해.
- [D-010] 브리지 기본 WS 포트를 `8080`에서 `9090`으로 변경.
  - 이유: 기존 환경의 8080 사용 충돌 회피.
- [D-011] 웹앱 실행 포트를 `3000`에서 `9091`로 고정.
  - 이유: 서버/로컬에서 포트 충돌 최소화 및 운영 단순화.
- [D-012] 웹앱 기본 WebSocket 주소를 DDNS 기반으로 변경(`ws://innert.iptime.org:28080`).
  - 이유: 고정 IP 의존을 줄이고 외부 접속 진입점을 통일.
- [D-013] 브리지에 첫 유효 UDP 패킷 1회 파싱 로그 추가(`LOG_FIRST_PACKET`).
  - 이유: 실환경 포맷 검증(141B/엔디안/status/activity)을 빠르게 수행하기 위해.
- [D-014] 3D 뷰를 포인트 중심에서 바닥 적층 타워 시각화로 운영.
  - 이유: 8x8 값 변화와 사람 움직임을 직관적으로 보기 위함.
- [D-015] 3D 바닥 평면 제거, 그리드만 유지.
  - 이유: 시인성 개선 및 불필요 시각 요소 축소.
- [D-016] 그리드 크기를 `xyScale`에 연동.
  - 이유: 데이터 영역과 바둑판 크기 불일치(여백) 문제 해결.
- [D-017] 카메라를 탑-측면 고정 + 좌우 회전만 허용(상하/줌/팬 제한).
  - 이유: 사용자 관찰 일관성 확보와 조작 복잡도 감소.
- [D-018] 사용자 대상 문구/응답은 한국어 기준으로 유지.
